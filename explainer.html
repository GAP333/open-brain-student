Under the Hood

Every piece and what it does

Six components, each with a clear job. Like a well-run office where everyone knows their role.

🗄️
Supabase
The Memory
A PostgreSQL database in the cloud that stores every thought. I own the data. It never disappears when an app shuts down or changes its business model.
🌐
Vercel App
The Capture Interface
A web app deployed to the internet with 7 ways to capture: type a note, record voice, paste a YouTube URL, upload a PDF, save a webpage, or search what's already stored.
✈️
Telegram Bot
The Pocket Capture
Send any message to a bot in Telegram and it saves instantly to the database. Works from my phone in two seconds. No app switching required.
🔌
MCP Server
The AI Bridge
An open-standard server that lets Claude (or any MCP-compatible AI) search and read my brain. Built on the Model Context Protocol — if I switch AI tools, I point them at the same server.
⚡
Enrichment Agent
The Tagger
A webhook-triggered function that runs every time a new thought is saved. It calls an AI, gets back tags, a category, and a one-sentence summary — all stored with the thought automatically.
📋
Weekly Digest
The Reporter
A scheduled job that runs every Sunday at 8am. It reads the last 7 days of captures, groups them by category, and writes a report on what I've been learning. Zero input from me.
Real Example

What happens when I capture something

Walking through what happens when I send a YouTube link to my Telegram bot.

1
I send a YouTube link to my Telegram bot
From my phone, I paste a YouTube URL into a conversation with my bot. Takes two seconds.
2
The bot extracts and saves the transcript
The Telegram bot edge function fetches the video transcript, formats it, and saves it to the Supabase database as a new thought. The bot replies "Saved to your brain."
3
The enrichment agent triggers automatically
A database webhook fires the enrich-thought function. Within seconds it calls OpenAI, gets back 3–5 tags, a category (like "learning" or "reference"), and a one-sentence summary. These are written back to the row.
4
It becomes searchable by Claude
Later, in Claude Desktop, I ask about something from that video. Claude calls the search_thoughts MCP tool, finds the transcript, and answers using my own captured content — not a generic web search.
5
It shows up in Sunday's digest
On Sunday at 8am, the weekly digest agent reads everything from the last 7 days including this transcript, groups it with other captures, and writes a summary of what I've been focused on that week.
